KEY DESIGN DECISIONS
====================

1. Modular Architecture
   - Each data source has dedicated parser
   - Consistent interface via BaseParser
   - Easy to extend with new sources

2. Ontology-First Approach
   - OWL ontology defines schema
   - Data mapped to ontology classes
   - Maintains semantic consistency

3. Mixed Download Strategy
   - Automatic downloads where possible
   - Manual downloads for licensed data
   - Clear documentation for all steps

4. Flexible Pipeline
   - Complete or partial builds
   - Source selection via CLI
   - Skip steps as needed
   - Caching support

5. Comprehensive Documentation
   - Multiple documentation files
   - Different audiences (users, developers)
   - Step-by-step instructions
   - Troubleshooting guides

USAGE EXAMPLES
==============

Basic Build (automated sources only):
  cd src
  python main.py --sources hetionet ncbigene

Full Build (with manual downloads):
  python main.py --sources hetionet ncbigene drugbank disgenet

Complete Build (with AOP-DB):
  python main.py \
    --sources hetionet ncbigene drugbank disgenet aopdb \
    --mysql-host localhost \
    --mysql-user root \
    --mysql-password pass

KNOWN LIMITATIONS
=================

1. Manual Downloads Required
   - DrugBank: Requires account approval (may take days)
   - DisGeNET: Requires account creation
   
2. Large File Sizes
   - AOP-DB: 7.2 GB compressed, 3+ GB uncompressed
   - Total raw data: ~10 GB
   
3. MySQL Dependency
   - AOP-DB requires MySQL Server (optional)
   
4. Data Fetching Errors
   - Some sources may be temporarily unavailable
   - Network timeouts possible for large files
   - Do not fabricate data - report errors honestly
